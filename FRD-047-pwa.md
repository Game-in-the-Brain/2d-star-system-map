# FRD-047: Progressive Web App (PWA)

**Project:** 2D Star System Map  
**Version Target:** 1.00  
**Priority:** P1  
**Depends On:** Core renderer pipeline — ✅ COMPLETE

---

## 1. Overview

The 2D Star System Map is converted into a Progressive Web App (PWA) so it can be:

- **Installed** on desktop and mobile home screens
- **Used offline** after the first visit (service worker precaches all assets)
- **Updated automatically** when new builds are deployed

---

## 2. User Story

> As a GM, I want to install the star system map to my device home screen and use it offline during game sessions without worrying about connectivity.

---

## 3. Technical Implementation

### 3.1 vite-plugin-pwa

The build pipeline uses `vite-plugin-pwa` to generate the service worker and manifest automatically:

```ts
// vite.config.ts
VitePWA({
  registerType: 'autoUpdate',
  workbox: {
    globPatterns: ['**/*.{js,css,html,ico,png,svg,json}'],
    maximumFileSizeToCacheInBytes: 5 * 1024 * 1024,
  },
  manifest: { /* see §3.2 */ },
})
```

### 3.2 Web App Manifest

| `orientation` | `any` |

| `start_url` | `/2d-star-system-map/` |
| `icons` | `icon-192x192.png`, `icon-512x512.png` (maskable) |

### 3.3 Service Worker

Generated by Workbox (`generateSW` mode):

- **Precaches** all production assets (JS, CSS, HTML, icons, manifest)
- **Navigation fallback** routes all paths to `index.html` (SPA behaviour)
- **Cleanup** removes outdated caches on activation
- **Auto-update** checks for new versions in the background

---

## 5. Service Worker Registration

`src/main.ts` registers the SW on boot via the `virtual:pwa-register` module provided by `vite-plugin-pwa`:

```ts
import { registerSW } from 'virtual:pwa-register';
registerSW({ immediate: true });
```

This ensures:
- The app works offline immediately after first load
- New deployments auto-update on the next visit

## 9. Version Tracking

This feature introduces the **0.01 increment version tracking system** documented in `VERSION.md`.

- Every `git commit` triggers the pre-commit hook
- Hook increments `VERSION` by `0.01` and auto-stages the file
- `vite.config.ts` reads `VERSION` and appends the git short hash
- Result: `1.05-a1b2c3d` style version strings
